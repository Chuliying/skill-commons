# Research subroutine

Load only after a material implementation decision has qualified Standard or Deep.
Factual requests stay on Skip and use repository inspection or host browsing.

## Contract

1. State the decision, research question, source budget, and stopping condition.
2. Prefer primary sources: specifications, official docs, source code, first-party APIs, and repo evidence.
3. Give decision-changing claims citations; label inference and uncertainty.
4. Stop when the decision is possible, the condition is met, or budget ends; report uncertainty.
5. Use a background agent only when parallel reading earns its cost; keep a synchronous fallback.

## Output boundary

Return findings to the active decision. `conditional durable output` exists only for
cross-session reuse: write a bounded `brainstorm.md` resolution or promote reusable
evidence to `docs/reference`. Add no Research mode, gate, or mandatory file.
